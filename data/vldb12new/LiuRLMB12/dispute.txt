For our VLDB'12 paper "Cologne: A Declarative Distributed Constraint
Optimization Platform", it was marked as "Build fails" by the team of
Collberg et al. at the University of Arizona on
http://reproducibility.cs.arizona.edu/.

From the notes, it shows that Collberg et al. compiled the code on
"Ubuntu12.04 64" OS. However, as clearly given by the online
documentation on the project website
http://netdb.cis.upenn.edu/cologne/documentation.html, the code should
only be compiled and run on "Ubuntu 10.04 32 bit" OS, with required
dependencies including Gecode 3.7.1.

Follow these step to compile and run the Cologne code.

(1) Get a Ubuntu 10.04 32-bit OS (use a virtual machine for
convenience), and install dependencies

# sudo apt-get install build-essential flex bison libssl-dev libgmp3-dev libboost-all-dev wget

(2) Install Gecode 3.7.1

# wget http://www.gecode.org/download/gecode-3.7.1.tar.gz
# tar xzvf gecode-3.7.1.tar.gz
# cd gecode-3.7.1
# ./configure
# make
# sudo make install

(3) Compile Cologne

# cd ~
# wget http://netdb.cis.upenn.edu/cologne/files/cologne_v0.1.tar.gz
# tar xzvf cologne_v0.1.tar.gz
# cd cologne_v0.1
# ./waf
